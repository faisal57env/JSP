    1. Check if a Number is Prime

    Spot the intruder divisors.

    Math + Loop

    Hook Me

    Imagine a VIP club with one strict rule: a true prime number allows entry only to 1 and itself. The moment any other number sneaks in as a clean divisor, the bouncer yells, “Fake prime!”

    Visual Anchor

    num = 29 check divisors from 2 to sqrt(29) 2 -> 29 % 2 = 1 ✅ 3 -> 29 % 3 = 2 ✅ 4 -> 29 % 4 = 1 ✅ 5 -> 29 % 5 = 4 ✅ No clean hit => PRIME Why stop at sqrt(n)? If n = a * b, then at least one of a or b <= sqrt(n)

    Code Breakdown

    PrimeCheck.java
    int num = 29;
boolean isPrime = num >= 2;
for (int i = 2; i * i <= num && isPrime; i++) {
    if (num % i == 0) {
        isPrime = false;
    }
}
System.out.println(isPrime ? "Prime" : "Not Prime");
    1. Line 1: Pick the suspect number.
    2. Line 2: Numbers below 2 are instantly disqualified.
    3. Line 3: Scan only till i * i <= num, the smart cutoff.
    4. Line 4-6: One perfect division is enough to destroy primality.
    5. Line 8: Print the verdict.

    Interactive Checkpoint

    Predict the output for num = 49. At which divisor does the loop catch it?

    Output: Not Prime. The catch happens at 7 because 49 % 7 == 0.

    2. Prime Numbers in a Range

    Run a prime radar over many numbers.

    Nested Loop

    Hook Me

    Visual Anchor

    Range: 10 to 20 10 -> divisible by 2 ❌ 11 -> no divisors ✅ 12 -> divisible by 2 ❌ 13 -> no divisors ✅ 14 -> divisible by 2 ❌ 15 -> divisible by 3 ❌ 16 -> divisible by 2 ❌ 17 -> no divisors ✅ 18 -> divisible by 2 ❌ 19 -> no divisors ✅ 20 -> divisible by 2 ❌

    if (prime) System.out.print(n + " ");
}
    1. Outer loop moves across the range.
    2. For each number, we reset the prime flag.

    4. Only survivors get printed.

    3. Armstrong Number

    Digits power up and report back.

    Digits

    Hook Me

    An Armstrong number is like a superhero team where every digit contributes its powered-up strength, and the total exactly rebuilds the original number.

    Visual Anchor

    Example: 153 Digits: 1, 5, 3 Power count = 3 digits 1^3 = 1 5^3 = 125 3^3 = 27 ----------- Total = 153 ✅ Original == powered sum => Armstrong

    Code Breakdown

    Armstrong.java
    int num = 153, temp = num, digits = 0, sum = 0;
while (temp > 0) {
    digits++;
    temp /= 10;
}
temp = num;
while (temp > 0) {
    int d = temp % 10;
    sum += Math.pow(d, digits);
    temp /= 10;
}
System.out.println(sum == num ? "Armstrong" : "Not Armstrong");
    1. First loop counts digits.
    2. Second loop extracts each digit with % 10.
    3. Each digit is raised to total digit count.
    4. Compare final sum with the original number.

    Interactive Checkpoint

    Is 370 Armstrong? Try the digit powers first.

    Yes. 3^3 + 7^3 + 0^3 = 27 + 343 + 0 = 370.

    6. Fibonacci using Recursion

    A problem that asks its smaller twins for help.

    Recursion

    Hook Me

    Recursion is like a chain of managers. Each manager says, “I can answer, but first ask two smaller managers.” The chain continues until tiny base cases know the answer instantly.

    Visual Anchor

    fib(5) ├─ fib(4) │ ├─ fib(3) │ │ ├─ fib(2) │ │ └─ fib(1) │ └─ fib(2) └─ fib(3) Base cases: fib(0)=0 fib(1)=1

    Code Breakdown

    FiboRec.java
    static int fib(int n) {
    if (n <= 1) return n;
    return fib(n - 1) + fib(n - 2);
}

System.out.println(fib(6));
    1. Base case stops the infinite fall.
    2. Every bigger call splits into two smaller calls.
    3. Results bubble back upward and combine.

    Interactive Checkpoint

    Answer: 3.

    7. Reverse a String using Recursion

    Peel from the front, rebuild from the back.

    String + Recursion

    Hook Me

    Imagine a stack of cards. You remove the top card again and again, then place them back after everything else. The order flips naturally.

    Code Breakdown

    ReverseRec.java

    11. Balanced String

    Brackets must open and close like matched doors.

    Stack

    Hook Me

    A balanced string is like borrowing books from a library. Every opening bracket checks out a book, and every closing bracket must return the exact correct one in reverse order.

    Visual Anchor

    Input: {[()]} Read '{' -> push { Read '[' -> push [ Read '(' -> push ( Read ')' -> pop ( ✅ Read ']' -> pop [ ✅ Read '}' -> pop { ✅ Stack empty => Balanced

    Code Breakdown

    BalancedString.java
    Stack<Character> st = new Stack<>();
String s = "{[()]}";
boolean ok = true;
for (char ch : s.toCharArray()) {
    if ("([{".indexOf(ch) != -1) st.push(ch);
    else if (ch == ')' && (st.isEmpty() || st.pop() != '(')) ok = false;
    else if (ch == ']' && (st.isEmpty() || st.pop() != '[')) ok = false;
    else if (ch == '}' && (st.isEmpty() || st.pop() != '{')) ok = false;
}
ok = ok && st.isEmpty();
System.out.println(ok ? "Balanced" : "Not Balanced");
    1. Open brackets go onto the stack.
    2. Closing brackets must match the latest opener.
    3. Wrong type or empty stack means failure.
    4. Final stack must be empty too.

    Interactive Checkpoint

    Does "([)]" pass?

    No. The closing ) expects (, but the stack top is [.

    16. 2nd Smallest Element without Sorting

    Track gold and silver in one pass.

    Array Logic

    Hook Me

    Visual Anchor

    arr = [7, 2, 9, 4, 1] smallest = INF, second = INF 7 -> smallest=7 2 -> second=7, smallest=2 9 -> ignore 4 -> second=4 1 -> second=2, smallest=1 Result: second = 2

    Code Breakdown

    SecondSmallest.java
    int[] arr = {7, 2, 9, 4, 1};
int first = Integer.MAX_VALUE, second = Integer.MAX_VALUE;
for (int x : arr) {
    if (x < first) {
        second = first;
        first = x;

        second = x;
    }
}
System.out.println(second);
    1. first stores smallest.
    2. second stores second smallest.
    3. If a new minimum arrives, the old minimum drops to second.
    4. The x > first check avoids duplicate confusion.

    19. Binary Search

    Cut the search space in half every move.

    Divide & Conquer

    Hook Me

    It is like guessing a word in a dictionary. You do not start from page 1. You jump to the middle, decide left or right, and shrink the world fast.

    Visual Anchor

    Sorted arr = [2, 5, 8, 12, 16, 23], target = 12 low=0, high=5 mid=2 -> 8 < 12 => move right low=3, high=5 mid=4 -> 16 > 12 => move left low=3, high=3 mid=3 -> 12 found ✅

    Code Breakdown

    BinarySearch.java
    int[] arr = {2, 5, 8, 12, 16, 23};
int target = 12, low = 0, high = arr.length - 1;
while (low <= high) {
    int mid = (low + high) / 2;
    if (arr[mid] == target) {
        System.out.println(mid);
        break;
    } else if (arr[mid] < target) low = mid + 1;
    else high = mid - 1;
}
    1. Works only on sorted arrays.
    2. Pick middle element.
    3. Discard half where target cannot exist.
    4. Repeat until found or range dies.

    Interactive Checkpoint

    21. Pascal Triangle

    Every interior number is born from two parents.

    Pattern + 2D Logic

    Hook Me

    Imagine a pyramid of bricks where every middle brick is made by merging the two bricks directly above it. The edges are always 1 because they have only one outer wall.

    Visual Anchor

    1 1 1 1 2 1 1 3 3 1 1 4 6 4 1 Rule: arr[i][0] = 1 arr[i][i] = 1 arr[i][j] = arr[i-1][j-1] + arr[i-1][j]

    Code Breakdown

    PascalTriangle.java
    int n = 5;
int[][] p = new int[n][n];
for (int i = 0; i < n; i++) {
    for (int j = 0; j <= i; j++) {
        if (j == 0 || j == i) p[i][j] = 1;
        else p[i][j] = p[i - 1][j - 1] + p[i - 1][j];
        System.out.print(p[i][j] + " ");
    }
    System.out.println();
}
    1. Only fill positions up to the current row index.
    2. Edges are always 1.
    3. Interior values come from two numbers above.
    4. Print row by row.

    Interactive Checkpoint

    What is the middle value in row 1 4 6 4 1?

    Answer: 6.

    23. Consecutive Frequency of Characters

    Run-Length Logic

    Hook Me

    Imagine a stadium crowd chant. Instead of writing every scream, you record the sound plus how many times it repeated in one uninterrupted burst.

    Visual Anchor

    Input: aaabbccccd Run 1: a a a -> a3 Run 2: b b -> b2 Run 3: c c c c -> c4 Run 4: d -> d1 Output: a3b2c4d1

    ConsecutiveFrequency.java
    String s = "aaabbccccd";
StringBuilder out = new StringBuilder();
for (int i = 0; i < s.length(); ) {
    char ch = s.charAt(i);
    int count = 0;
    while (i < s.length() && s.charAt(i) == ch) {
        count++;
        i++;
    }
    out.append(ch).append(count);
}
System.out.println(out);
    1. Pick the current character.
    2. Count how long that run continues.
    3. Append character plus frequency.
    4. Move to the next new character block.
